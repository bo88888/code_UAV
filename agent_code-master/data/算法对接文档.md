# 低空算法服务对接协议

所有算法通过 `POST /infer` 接收统一 MCP JSON：

```json
{
  "task_id": "MISSION_001",
  "subtask_id": "ROUTE_MED_01",
  "tool_name": "medical_time_window_scheduler_service",
  "input_data": {
    "mission": {
      "mission_type": "emergency_medical",
      "delivery_points": [],
      "cargo_weight_kg": 2.5,
      "priority": 4,
      "environmental_constraints": {}
    },
    "previous_results": {
      "WEA_01": {"is_flyable": true},
      "UAV_01": {"available_uavs": []}
    }
  },
  "parameters": {},
  "output_schema": ["uav_id", "eta", "time_window_met"]
}
```

统一响应：

```json
{
  "subtask_id": "ROUTE_MED_01",
  "tool_name": "medical_time_window_scheduler_service",
  "success": true,
  "output": {
    "uav_id": "UAV-MED-001",
    "eta": "2026-06-11T10:15:00",
    "time_window_met": true
  },
  "message": "TWA-MILP optimal schedule generated"
}
```

## 工具清单

| 阶段 | 工具 |
| --- | --- |
| A 环境检查 | `airspace_check_service` |
| A 环境检查 | `weather_check_service` |
| A 环境检查 | `uav_status_service` |
| A 环境检查 | `dock_status_service` |
| B 路线调度 | `compliance_route_service` |
| B 路线调度 | `weather_adaptive_dispatch_service` |
| B 路线调度 | `medical_time_window_scheduler_service` |
| B 路线调度 | `agentic_task_allocation_service` |
| C 风险评估 | `risk_assessment_service` |
| R 报告 | `dispatch_report_service` |

航线算法必须从 `input_data.previous_results` 读取气象、空域、无人机和
机巢检查结果，不能绕过上游约束。
